# リズム戦闘PoC 引き継ぎ補足資料

最終更新: 2026-06-20

> 本資料は検証経緯を残すための補足資料である。現在仕様の正本は[`../13_rhythm_audio_data.md`](../13_rhythm_audio_data.md)とし、差異がある場合は正本を優先する。

## 1. 目的

GPSゲーム本体へリズム戦闘を導入できるか検証するため、Web Audio APIだけで動作する独立HTML版PoCを作成した。主な検証対象は、音楽と攻撃判定の同期、スマートフォンでの操作性、端末遅延補正、譜面別ゲームバランスである。

現時点ではゲーム本体へ未統合であり、バックエンド、DB、ログイン、位置情報には依存しない。

## 2. 主要ファイル

- `rhythm-battle-poc.html`: PoC画面
- `css/rhythm-battle-poc.css`: PC・スマホ表示

## 3. 現在の楽曲と譜面

### 楽曲

1. ストレート・バトル
2. シンコペーション・ドライブ
3. シャッフル・クエスト
4. 余白のステップ
5. ミッドナイト・ブレイク（JAZZ）

### 譜面

1. 基本
2. 裏拍
3. 技巧
4. 余白
5. ブレイク技巧

全曲8小節、開始前に4分音符4拍のカウントを行う。BPMは80から180、初期値126。曲と譜面は任意の組み合わせで選択できる。

## 4. ジャズの仕様

- 進行は`Dm7 -> G7 -> Cmaj7 -> A7(b9)`を2周する。
- 4分音符のランニングベースを32拍連続で演奏する。
- ピアノは各小節頭に約1小節のコードを鳴らす。
- ジャズではキック、スネア、ハイハット、ライド等の打楽器を使用しない。
- ブレイク技巧譜面は前半を密にし、長い休符後に復帰する。
- スマホ向け調整として、ベースピーク0.20、ピアノ0.075、ベースフィルター700Hz、第2倍音36%、第3倍音12%を使用する。

## 5. 判定と入力

- 音声、判定、表示ノート、ヒントは同じ`note.time`を参照する。
- 入力は`click`ではなく`pointerdown`で取得する。
- `event.timeStamp`を使い、イベント処理開始時刻ではなく実際の押下時刻へ戻して判定する。
- PERFECTは`±80ms`、GOODは`±180ms`、範囲外はMISS。
- MISS時に次のノートを消費しないため、一度の失敗で譜面位相がずれない。
- PERFECTは18ダメージ・1200点、GOODは10ダメージ・600点、アクセントは6ダメージ追加。

## 6. タイミング調整

### PoCで実装済み

- ヘッダーの「タイミング調整」からユーザーが任意に実行できる。
- 2拍を練習、続く8拍を計測し、中央値を補正値とする。
- 補正値は`±250ms`へ制限する。
- `localStorage`の`rhythmBattleTimingCalibration`へバージョン付きで保存する。
- 次回起動時に自動適用し、毎回のキャリブレーションは要求しない。
- 保存できない環境では現在のセッションだけ有効。

### ゲーム本体向け方針のみ。未実装

- ゲーム設定と戦闘開始前設定の両方から能動的に再調整できるようにする。
- 現在値、最終調整日時、再調整、リセットを提供する。
- 直近12件以上の有効タップで中央値が同方向へ40ms以上、75%以上が同方向なら、戦闘終了後に再調整を提案する。
- 補正値は自動変更しない。「あとで」を選んだ場合は7日間再提案しない。
- 補正値と判定履歴はユーザーDBへ保存せず、端末ローカルに保持する。
- Bluetooth等の出力経路別プロファイルは将来検討。

## 7. ヒント機能

- 初期値ON。戦闘開始前にON/OFFを選択する。
- 選択譜面の各`note.time`に高いルート音と短いクリックを鳴らす。
- 通常ヒントは伴奏の奥へ下げ、三角波ピーク0.075、クリック0.045。
- キャリブレーション音は聞き取りやすさ優先で0.30、0.24を維持する。

## 8. 戦闘バランス

- 敵最大HPは、選択譜面を全PERFECTした理論最大ダメージの70%を切り上げて決定する。
- 基本・裏拍・技巧・余白は全PERFECTなら22拍目前後から撃破可能。
- ブレイク技巧は前半最大420ダメージ、敵HP429。長いブレイク後の復帰フレーズから撃破できる。
- 8小節終了時にHPが残っていれば「時間切れ」と残りHPを表示する。
- 撃破時はレーン中央に「撃破！」、スコア、コンボを表示する。

## 9. 再生ライフサイクル

- 戦闘開始時に予約済み音源、タイマー、描画、スコア、HPを完全にリセットする。
- 敵撃破または8小節終了で音源を停止する。
- 未撃破でも曲はループせず、戦闘開始を押すと先頭から再開する。

## 10. 表示

- PCは上段にPoC表記・タイトル・説明、下段に設定を置く2段構成。
- スマホは曲、譜面、BPM、ヒント、戦闘開始を一行に配置する。
- レーン高は`100dvh`基準で伸縮し、スコア4項目までを最初の画面へ収める。
- 高さ600px以下ではタイトルと敵画像を省略する。
- 撃破・時間切れ・キャリブレーションはレーン内オーバーレイで表示する。
- 音の有無にかかわらず、レーン下端の4区画ガイドで小節内の現在拍を常時表示する。
- 各4分音符の拍頭で判定線を120ms発光させる。ガイドと発光は`AudioContext.currentTime`、曲開始時刻、BPMから算出する。
- 表拍、裏拍、スイングのノーツはすべて22px四方。アクセントはサイズを変えず、輪郭と発光で区別する。
- `prefers-reduced-motion: reduce`では区画の拡大を行わず、明度と輪郭だけを切り替える。

## 11. 実行と検証

```powershell
node --test server\tests\rhythm-battle-poc.test.js
node --check js\rhythm-battle-poc.js
```

配布版は`dist/rhythm-battle-poc/index.html`を開く。外部ライブラリ、画像、API、DBは不要。

HTMLのキャッシュ番号はCSS `v=11`、JavaScript `v=16`。

2026-06-20の視覚ビートガイド追加後は45テスト全件成功、JavaScript構文エラーなし。

## 13. 注意事項

- PoC本体、CSS、JavaScript、テスト、仕様書、引き継ぎ資料は視覚ビートガイド実装時にGit管理へ追加した。`dist/`などの配布物は未追跡の場合があるため、別チャットで作業を始める前に`git status --short`を確認すること。
- 音量・音色の自動テストは設定値と構成を検証するものであり、最終的な聴感は実機確認が必要。
- 本体向け自動再調整サジェストはドキュメント方針のみで、PoCにもゲーム本体にも未実装。
