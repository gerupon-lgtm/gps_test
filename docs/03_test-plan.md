## 1.1 現行版で追加すべきテスト観点

本書はPoC開始時のテスト計画を残す。現行版では以下も確認対象に含める。

- Cookie認証、ログイン/ログアウト、ログイン停止時の挙動
- サーバー権威の戦闘開始/行動/復帰、戦闘中アイテム使用、戦闘終了後の自動復帰
- 戦闘中の「にげる」。報酬・撃破履歴・戦闘不能ペナルティが付かず、探索画面へ戻ること
- レベルアップ、毒、称号、マーケット出品などのトースト表示
- 宿屋/道具屋の10mチェックイン、範囲外ボタン制御、売買確認
- マーケットの出品/購入/取消、手数料負担、手数料台帳、所持金/在庫更新
- スポット一覧での「撃破済み」バッジ、クールダウン分数表示、距離順表示
- 管理アプリのCSV入出力、SJIS/UTF-8、直接登録、近接チェック、撃破済み編集
- 今後追加するNPCでは、ランダム出現、出現期限、回復/販売効果、位置共有/遭遇通知のクールダウンを確認する

---

## 2. テスト目的

GPS連動ブラウザゲームの検証版について、以下が期待通り動作することを確認する。

- GitHub Pages上でHTTPS配信できること
- スマートフォンブラウザで位置情報を取得できること
- CSVファイルを読み込めること
- 現在地と登録地点の距離判定ができること
- 範囲内で敵が出現すること
- 戦闘、勝利報酬、敗北ペナルティが動作すること
- localStorageに状態が保存されること

## 4. 事前準備

1. GitHubリポジトリを作成する。
2. GitHub Pagesを有効化する。
3. HTTPSでアクセスできることを確認する。
4. `data/spots.csv` を配置する。
5. 検証地点を1〜3件登録する。
6. スマートフォンで位置情報の利用を許可する。
7. 実際に移動しても問題ない安全な地点を選ぶ。

## 7. 実地検証時の注意

- 歩きスマホを前提にしない。
- 道路上、線路付近、私有地、危険区域を検証地点にしない。
- 検証地点は公園、広場、建物入口付近など安全な場所にする。
- 深夜や人通りの少ない場所での検証は避ける。
- GPS精度が悪い場合は、屋外で再確認する。
- 個人宅や学校などプライバシー上問題のある地点を登録しない。

---

## 8. 合格基準

以下をすべて満たせば、検証成功とする。

- HTTPS環境で位置情報が取得できる。
- CSVのスポット情報を読み込める。
- 登録地点との距離判定ができる。
- 範囲内で敵が出現する。
- 戦闘の勝敗処理ができる。
- 勝利時にアイテムが保存される。
- 敗北時にペナルティが保存される。
- ページ再読み込み後も状態が保持される。
