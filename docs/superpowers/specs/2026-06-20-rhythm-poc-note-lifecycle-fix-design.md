# リズム戦闘PoC ノーツ表示寿命修正設計

作成日: 2026-06-20

## 問題

MISS判定ではデータ上の`note.missed`だけが更新され、既に生成されたノーツDOMが削除されない。以後の描画対象から外れるため、DOMは最後の位置で停止する。撃破時にも未判定ノーツが残り、ビートガイドだけが進んでいるように見える。

## 修正

- HIT時はノーツDOMを削除し、`state.noteEls`からも参照を削除する。
- MISS確定時も同じ削除処理を行う。
- 撃破時は残っているノーツDOMとMapを全消去する。
- 時間切れと戦闘リセットも同じ全消去処理を共有する。

## 対象外

- 音声時計、視覚ビート計算、判定窓、譜面、スコア、rAFウォッチドッグの変更

## 検証

- ソース契約テストでHIT、MISS、撃破の全経路から削除処理が呼ばれることを確認する。
- JavaScriptキャッシュを`v=21`へ更新し、配布フォルダを同期する。
