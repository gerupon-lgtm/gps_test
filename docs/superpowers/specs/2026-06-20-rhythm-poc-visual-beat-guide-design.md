# リズム戦闘PoC 視覚ビートガイド設計

作成日: 2026-06-20

## 目的

音を出せない環境でも、追加設定なしでリズム戦闘を成立させる。音ありと無音で異なるルールへ分岐させず、同じ譜面、判定、得点、難易度を使う。

## 採用する表示

現行レーンへ次の2要素を常時表示する。

1. レーン下端に4分音符4拍分のガイドレールを置き、小節内の現在拍を左から順に点灯する。
2. 各4分音符の拍頭で判定線を短く発光させる。レーン全面の強い明滅は行わない。

開始前の4カウントは既存の大きな `1`～`4` 表示を維持する。カウント中もガイドレールを同じ順序で点灯し、曲開始後は第1拍へ自然につなぐ。

音あり、ゲーム内音量0、OS側の消音を区別せず、すべて同じ表示とする。ブラウザから実際の可聴状態を確実に取得できないため、無音の自動判定には依存しない。

## ノーツ表示

現行サイズを次のように縮小する。

- 表拍、裏拍、スイング: すべて22px四方
- 中心位置は現在と同じとし、半サイズ分の負のマージンを設定する。
- アクセントはノーツ自体を大型化せず、既存の縁色または短い発光で区別する。

小型画面でも同じ基準サイズを使う。ノーツをタップ対象にはせず、入力対象は従来どおり「こうげき」ボタンとレーンであるため、タッチターゲット寸法には影響しない。

## 時刻の基準

視覚ガイドは、音声、判定、ノーツ表示と同じ `AudioContext.currentTime`、`state.startTime`、BPMを参照する。CSSだけの独立した無限アニメーションは使わない。

曲中は次の値を描画フレームごとに求める。

- 曲開始からの経過拍: `songTime / beatSeconds(bpm)`
- 小節内の拍番号: `floor(elapsedBeats) % 4`
- 拍内進行度: `elapsedBeats - floor(elapsedBeats)`

ガイドレールは拍番号に対応する1区画だけを点灯する。判定線の発光は拍内進行度の先頭付近だけ有効にし、常時点滅や判定ノーツへの追従は行わない。

4カウント中は、既存の `scheduleCountIn()` が使う各カウント時刻と同じ時刻でガイドレールを更新する。戦闘停止、撃破、時間切れ、再開始では点灯状態と発光状態を初期化する。

## ゲーム性

- 視覚ガイドは四分音符の基準拍だけを示す。
- 裏拍、スイング、シンコペーションなど、実際に押すタイミングそのものを追加表示しない。
- 落下ノーツは従来どおり各 `note.time` を示す。
- 判定幅、ダメージ、得点、敵HP、譜面、ヒント音、キャリブレーションは変更しない。

これにより無音時にテンポと小節位置は維持しやすくなるが、譜面を読む技能は残る。

## 表示とアクセシビリティ

- 現在拍は色だけでなく、明度と区画の拡大または輪郭でも区別する。
- 判定線の発光は短く、画面全体を明滅させない。
- `prefers-reduced-motion: reduce` では拡大アニメーションを省き、瞬時の明度・輪郭変更だけにする。
- ガイドは装飾要素として扱い、スクリーンリーダーへ拍ごとのライブ通知は行わない。
- ガイドレールは判定線や落下ノーツと重ならないレーン下端へ置く。

## 実装単位

- HTML: レーン内に4区画のガイドレールを追加する。
- JavaScript: BPMと曲内時刻から現在拍・拍内進行度を返す純粋関数を追加し、描画処理から表示クラスまたはCSSカスタムプロパティを更新する。
- カウントイン: 既存タイマーからガイドの拍番号を更新する。
- CSS: ガイドレール、判定線発光、縮小ノーツ、低モーション設定を追加する。
- ライフサイクル: リセット・停止・終了時に視覚ガイドを待機状態へ戻す。

## テスト

Node.jsテストへ次を追加する。

- BPMから拍番号と拍内進行度を正しく算出する。
- 第4拍から次小節の第1拍へ戻る。
- 拍境界の直前・直後で発光状態が正しく切り替わる。
- 戦闘の既存譜面、判定、得点、HP計算に変更がない。
- HTMLに4区画のガイドが存在する。
- CSS上のノーツ寸法が拍種別によらず22pxで統一されている。

最終確認では `node --test server\tests\rhythm-battle-poc.test.js` と `node --check js\rhythm-battle-poc.js` を実行し、PC幅とスマートフォン幅でレーンの余白、ガイドの視認性、ノーツとの非干渉を目視確認する。

## 対象外

- OS音量、消音スイッチ、Bluetooth出力状態の検出
- 無音時だけの得点・判定・敵HP調整
- プレイヤーへ無音モード選択を要求する設定
- 振動や端末ハプティクス
- ゲーム本体への統合
