# リズム戦闘PoC 合成表示正式採用・二小節カウント設計

## 目的

実機検証で最もプレイアビリティが高かった合成アニメーション表示を標準動作にする。同時に、戦闘開始前のカウントを一小節から二小節へ拡張し、プレイヤーがテンポをつかむ時間を増やす。従来のrAF表示は比較・復旧用のサブモードとして残す。

## カウント仕様

カウントは4/4拍子の二小節、合計8拍分とする。

| 小節 | 発音・表示位置 | 画面表示 |
| --- | --- | --- |
| 1小節目 | 1拍目、3拍目 | `1`、`2` |
| 2小節目 | 1拍目、2拍目、3拍目、4拍目 | `1`、`2`、`3`、`4` |

イベント列は四分音符単位で `[1, 休, 2, 休, 1, 2, 3, 4]` とする。楽曲、判定、ノーツ、拍ガイドの開始時刻は、カウント開始から8拍後とする。カウント中の攻撃不可、`START!`表示、各カウントの拍ガイド発光は既存仕様を維持する。

## 表示モード

- 通常URLでは合成アニメーション表示を使用する。
- `?visual=raf`指定時だけ従来のrAF表示を使用する。
- `?debug=1`は選択された表示モードと併用できる。
- Web Animations API非対応環境では、通常URLでも自動的にrAF表示へフォールバックする。
- 旧`?visual=compositor`は互換性のため合成表示として受け付ける。

表示モードだけを切り替え、音声スケジューラ、タップ時刻取得、判定、ダメージ、譜面には変更を加えない。

## 保管と配布

変更前のv24配布一式を日時入りZIPとして`dist/archive/`へ保存し、SHA-256をメイン資料へ記録する。新配布版は従来どおり`dist/rhythm-battle-poc/`へ同期する。手動アップロード対象は`index.html`、CSS、JavaScriptとする。

## テスト

- 二小節カウントの総時間が8拍であること。
- カウントイベントが`1, 2, 1, 2, 3, 4`の順で、指定拍に配置されること。
- 通常URLと旧`?visual=compositor`が合成表示になること。
- `?visual=raf`がrAF表示になること。
- Web Animations API非対応時にrAFへ戻ること。
- 既存の音声、判定、譜面、描画寿命テストがすべて維持されること。

## 対象外

- プレイヤー向け表示モード設定UI
- カウント音色や音量の再設計
- 楽曲・譜面・戦闘バランスの変更
