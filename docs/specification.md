# GPS連動ブラウザゲーム 検証版 仕様書

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | GPS連動ブラウザゲーム 検証版 仕様書 |
| 対象フェーズ | PoC / 技術検証 |
| 配信環境 | GitHub Pages |
| データ管理 | 同一リポジトリ内CSVファイル |
| 作成日 | 2026/06/13 |

---

## 2. 検証の目的

本検証では、スマートフォン等のブラウザから取得したGPS情報を利用し、あらかじめCSVに登録された地点情報との距離判定を行い、指定範囲内に入った場合に敵キャラクターを出現させ、簡易戦闘・報酬取得・敗北時ペナルティ待機までの一連の流れが実現可能か確認する。

---

## 3. 前提条件

| 項目 | 前提 |
|---|---|
| 配信方式 | GitHub PagesによるHTTPS配信 |
| 実行環境 | スマートフォンのWebブラウザを主対象 |
| 位置情報取得 | ブラウザのGeolocation APIを使用 |
| データ配置 | GitHubリポジトリ内にCSVファイルを配置 |
| DB | 使用しない |
| ログイン | 検証版では実装しない |
| セーブデータ | localStorageを使用 |
| 地点作成 | Google Maps APIを使った別アプリで作成予定 |
| 地図表示 | 検証版では必須としない |

---

## 4. 検証対象範囲

### 4.1 対象に含めるもの

- 現在地の取得
- GPS利用許可の確認
- 現在地の緯度・経度・精度表示
- CSVファイルの読み込み
- 登録地点との距離計算
- 指定範囲内に入った場合の敵出現
- 簡易戦闘
- 勝利時のアイテム取得
- 敗北時のペナルティ待機
- localStorageへの状態保存
- GitHub Pagesでの動作確認

### 4.2 対象に含めないもの

- ユーザー認証
- サーバーDB
- ランキング
- 課金
- 本格的な不正対策
- 複数端末間のセーブデータ同期
- 管理画面
- Google Maps APIを使った地点作成機能
- 本格的な戦闘バランス調整

---

## 5. 想定ユーザーフロー

1. ユーザーがGitHub Pages上のゲームURLにアクセスする。
2. 画面に位置情報利用の説明が表示される。
3. ユーザーが位置情報取得を許可する。
4. ブラウザが現在地を取得する。
5. アプリがCSVファイルを読み込む。
6. 現在地と登録地点の距離を計算する。
7. 指定範囲内の地点があれば敵キャラクターを表示する。
8. ユーザーが戦闘を開始する。
9. 勝利した場合、アイテムを取得する。
10. 敗北した場合、指定時間の再戦待機状態になる。
11. 再戦可能時間を過ぎると、再び戦闘できる。

## 8. 機能要件

### F-001 位置情報取得

ブラウザのGeolocation APIを使用し、現在地を取得する。

| 項目 | 内容 |
|---|---|
| 取得方式 | watchPosition |
| 表示項目 | latitude / longitude / accuracy |
| エラー時 | 位置情報未許可、取得失敗、タイムアウトを表示 |

### F-002 CSV読み込み

同一リポジトリ内のCSVファイルをfetchで読み込む。

| 項目 | 内容 |
|---|---|
| 読み込み対象 | spots.csv / enemies.csv / items.csv |
| 読み込みタイミング | 初期表示時 |
| 失敗時 | エラーメッセージ表示 |

### F-003 距離判定

現在地と地点座標の距離を計算する。

| 項目 | 内容 |
|---|---|
| 計算方式 | Haversine式 |
| 判定条件 | 距離 <= radius_meters |
| 精度条件 | accuracy <= 許容GPS精度 |

### F-004 敵出現

範囲内に入ったスポットに紐づく敵を表示する。

| 項目 | 内容 |
|---|---|
| 出現条件 | 距離判定OK、active=true、ペナルティ中でない |
| 複数該当時 | 最も近いスポットを優先 |
| 表示内容 | 敵名、スポット名、戦闘開始ボタン |

### F-005 簡易戦闘

検証版では簡易ターン制とする。

| 項目 | 内容 |
|---|---|
| プレイヤーHP | 100固定 |
| 攻撃方式 | ボタン押下で1ターン進行 |
| ダメージ | 検証用の簡易計算 |
| 勝利条件 | 敵HPが0以下 |
| 敗北条件 | プレイヤーHPが0以下 |

### F-006 報酬取得

勝利時にスポットに紐づくアイテムを取得する。

| 項目 | 内容 |
|---|---|
| 保存先 | localStorage |
| 保存内容 | item_id、spot_id、取得日時 |
| 重複取得 | 検証版では許可 |

### F-007 敗北ペナルティ

敗北時に再戦可能時刻を保存する。

| 項目 | 内容 |
|---|---|
| 保存先 | localStorage |
| 保存単位 | spot_id単位 |
| 待機時間 | spots.csvのpenalty_minutes |
| 再戦条件 | 現在時刻 >= 再戦可能時刻 |

## 10. 検証成功条件

- GitHub Pages上でHTTPS配信できる。
- スマートフォンブラウザで位置情報許可ダイアログが表示される。
- 現在地の緯度・経度・精度が表示される。
- CSVからスポット情報を読み込める。
- 登録地点との距離が表示される。
- 指定半径内で敵が出現する。
- 戦闘を実行できる。
- 勝利時にアイテムが保存される。
- 敗北時に再戦待機が保存される。
- ページ再読み込み後もアイテム・ペナルティ状態が維持される。

---

## 11. 検証上の注意

- CSVファイルはGitHub Pagesで公開されるため、非公開にしたい座標や個人情報を含めない。
- GitHub Pagesは静的サイト向けであり、PHP、Python、Rubyなどのサーバーサイド処理は使えない。
- Geolocation APIはHTTPS環境またはlocalhostなどの安全なコンテキストが前提。
- GPS精度は端末、場所、建物、天候、設定により変動する。
- 実在地点を使う場合、私有地、危険区域、交通量の多い場所を避ける。
